A link to my "vision board" about how this site will look.

A fantasy quest board website. I imagine this as having a main page with short thumbnails, payouts, danger level, and quest titles on cards for adventuring parties to browse through. clicking on these cards would bring you to a page containing further details/descriptions for the quest including a user/poster of the quest and a pin on a map as to the location of the quest. Expanding this map will allow you to enter the name of or click a starting node and will using A* or some other pathfinding to create a trail from that node to the node nearest to the pins coordinates. I'd also like to add regional weather effects that affect the priority/travel difficulty of a group of nodes, monster sightings that affect a specific nodes and certain holiday events that may make travel around a certain node take longer/be less prioritized. 

If I am being honest this is mainly something that just sounds really fun to create and I believe could act as an impressive showing to potential employers. However with a few tweaks to the idea I believe this could be an app for dungeon masters and players of table top role playing games if I were to either change it to take manual settings from the user or give it a mode to do so those who like myself get really into world design could import an image of their maps and place nodes at major locations on their maps with some more imbetween for the paths they travel. I imagine in this scenario that the site would have session storage to remember what you're doing as you move around the site and save and load buttons to provide a file to do those actions. The user would be in this scenario able to set up the following:
    Map nodes
    Quest giver profiles
    individual quest details including descriptions, payout, difficulty, location, coordinates and more
    Node traversal difficulty
    Region setup
    selecting weather for a region
    terrain for nodes
    additional travel difficulty
This could be useful for a Dungeon master particularly during prep to keep them from re-using the same pathways and hopefully push them to coming up with new encounters and events in different regions of the map. Say normally a party would travel through the forest as a direct path but bandits make traveling that way more difficult it could route the players through a town to the north as a preferred path which could spice up the DMs preparations. If I was to follow this particular thread of thought I believe adding buttons to randomize weather and randomize the location and presence of monsters could facilitate creativity. In shortest terms I want to make a mapping app for role playing games that acts as a creative tool for dungeon masters. (Could be fun to by default have a map with a crystal ball theming like the players are peering into it to find out where to go.)

If you can't tell I just start writing and my brain expands the idea as I type.

not sure what you mean by technologies but I expect to be creating a session storage to hold the work together while the quests and map are being changed. I intend for the user to be able to press a button to download a file holding page data. I intend to modify CSS based on screen size. I need to allow for the uploading of files somehow. I will be using classes to store data about location nodes, regions, quests, and quest givers. I believe I will be converting some JSON into javascript objects or something similar. These are some of the "technologies" I intend to use.
